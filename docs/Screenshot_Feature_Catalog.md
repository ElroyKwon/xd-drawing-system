# Screenshot Feature Catalog

ACC(Autodesk Construction Cloud / Forma) 벤치마크 캡처 53장(`스크린샷/`, `fa1872d`로 git 추적)을 기능 단위로 카탈로그화한 문서다.

- 목적: **"스크린샷에 보이는 기능을 전부 구현"**이라는 프로젝트 목표의 단일 백로그 source of truth.
- 사용법: 각 행이 미구현 기능 후보다. 여기서 우선순위를 뽑아 feature-note → planning-gate → ai-loop 메타 프롬프트로 내려보낸다.
- 작성: 2026-06-23, 캡처 4배치를 병렬 분석해 합산.
- 표기: `구현됨`(슬라이스 존재) / `부분`(셸·골격만, 데이터·인터랙션 미구현) / `미구현` / `범위 외`(ACC 제품 UI 아님).

현재 구현 기준선: Hub(My Home·프로젝트·프로젝트템플릿 셸), 프로젝트 작성 모달, 프로젝트 목록, Project Admin 셸, Build 셸, 로컬 2D 시트뷰어 셸.

## 우선순위 후보 (제안)

스크린샷이 가장 깊게/반복적으로 담은 영역 = 사용자가 중요하게 본 영역으로 해석.

1. **2D 시트 뷰어 마크업 도구군** (E·F·G·H, 25+장) — 텍스트·클라우드·폴리라인·도형·다각형 마크업, 속성 패널, 마크업 로그, 측정/교정, 시트 비교, 마크업→이슈 연결. 가장 비중 큼.
2. **Build 시트 목록** (D) — 버전 세트·공종·태그 테이블, 검색/필터, 행 메뉴. 뷰어 진입의 전제.
3. **프로젝트 템플릿 전체 워크플로우** (B, 13장) — 생성 2단계 모달, 샘플 갤러리, 템플릿 상세(구성·멤버·회사·알림 매트릭스), 작성 모달 템플릿 연동.
4. **Build 프로젝트 홈 위젯** (C) — 진행률·빠른링크·날씨·작업·종합 분석 차트.
5. **Build 파일(Files) 문서관리** (I) — 폴더 트리, 업로드 모달.
6. **My Home 위젯** (A) — 할당·책갈피·최근 항목·온보딩.

> ⚠ 일부 기능(실제 마크업 영속화, 파일 업로드/저장, 시트 변환·렌더, Autodesk 연동)은 AGENTS.md의 HUMAN_GATE 대상이다. 카탈로그는 "보이는 기능" 목록일 뿐, 외부 연동·영속화·유료 SDK·렌더 엔진 채택은 별도 승인 게이트를 거친다.
