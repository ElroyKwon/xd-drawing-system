## Stage goal

기존 `src/ProjectAdminView.tsx` 셸을 재사용해 **템플릿 상세** 컨텍스트를 카탈로그(§B 173409~173637) 구조 수준으로 구현한다. 진입은 M1의 허브 템플릿 테이블 행 클릭. "없는 화면 만들기"가 아니라 캡처가 기록한 영역·컨트롤·빈상태·탭·매트릭스의 갭 메우기.

## Co-design log (frozen decisions — 2026-06-24)

0. **상호작용 깊이 = 구조 우선 (M1 결정 1 계승).** 살아있는 로컬 state로 동작해야 하는 것: 모드 분기, 측면 네비 distinct 전환, 알림 매트릭스 그룹 전개/접기, 모달 열고닫기. 정적 affordance(영속화/실연동 없음): 주파수 드롭다운 값, 권한 토글바, 게시 토글, 검색/페이저/그룹작성.
1. **범위 = 템플릿 상세 우선.** 캡처 근거가 강한 템플릿 상세 5화면(구성·템플릿 구성원·프로젝트 구성원·회사·알림 매트릭스)을 카탈로그 수준으로. 일반 프로젝트 어드민의 **캡처 없는 항목(브리지·액티비티·위치·설정)은 현 플레이스홀더 유지**(추정 구현 금지).
2. **알림 매트릭스 = 전체 도구 충실도.** 보조 네비(알림 그룹 작성·기본 알림 설정·Opt out) + "필요한 작업 알림"/"기타 알림" 그룹 + 15개 도구 + 주파수 드롭다운 + 권한 토글바 + 전개/접기.
3. **진입 동선 = 허브 템플릿 행 클릭.** M1 허브 템플릿 테이블 행 클릭 → `ProjectAdminView`를 `mode='template'`로 렌더(셸 재사용). 일반 프로젝트 모드는 회귀 없이 유지.

- **시드 필요**: 현 `hubTemplates`는 기본 빈 배열이라 `template-row`가 없다. 검증 가능하게 하려면 **시드 허브 템플릿 1건 이상**을 `App.tsx`(또는 `projectAdminData.ts`)에 추가해 행을 기본 렌더한다.
- M1 허브 템플릿 테이블 행(`data-testid="template-row"`)에 클릭 핸들러 추가 → 선택 템플릿 식별자를 App state(`selectedTemplateId`)에 담는다.
- **App state 분기(명시)**: `activeView` 유니온에 `"template-admin"`을 추가하고, 그 분기에서 `<ProjectAdminView mode="template" templateName={...} onBackToProjects={() => setActiveView("project-templates")} />`를 렌더한다. **기존 `project-admin` 분기·`project` prop·`onBackToProjects` 시그니처는 불변**(일반 모드 호출부 무변경 → A9 회귀 격리).

### 5개 섹션 화면 (캡처 근거)
1. **구성**(173421): 액션바 `[+프로젝트 만들기][사본 작성][보관]` + "일반" 행(템플릿 이름 + 값 + 오른쪽 끝 편집 연필) + "고급" 행(템플릿 게시 토글 OFF=「아니요」/ON=「예」, 설명문 "이 템플릿은 프로젝트를 작성할 수 있는 허브의 모든 구성원이 사용할 수 있습니다."가 토글 왼쪽).
2. **템플릿 구성원**(173409): 제목+설명 + `[+템플릿 구성원 추가]` + 검색 + 테이블(구성원·이메일·회사·역할·액세스 레벨) + 페이저("N개 중 M~K개 표시 중" + ≪‹ 1/1 ›≫). 캡처 실값: 회사="TEST-", 역할="관리자", 액세스="Project Admin".
3. **프로젝트 구성원**(173442): `[+프로젝트 구성원 추가]` + 검색 + 헤더 5컬럼(프로젝트 구성원·이메일·회사·역할·액세스 레벨) + **빈상태 확정**(안전모 일러스트 + "표시할 프로젝트 구성원이 없습니다.") + 설명문 "이 템플릿에서 작성된 프로젝트에 포함될 구성원을 관리합니다."
4. **회사**(173455): `[+회사 추가]`(+우측 info) + 검색(+좌측 info) + 테이블(이름·업종·추가된 일시 + 행 케밥 ⋮).
5. **알림**(173517~173637): 좌측 보조 네비(`[+알림 그룹 작성]` + "기본 알림 설정"(활성 체크) + "Opt out") + 우측 매트릭스 표(아래).

### 알림 매트릭스 (173517~173637) — 검토 차단 1·2·3 정정 반영
- 제목 "프로젝트 알림 설정". 표 헤더: **도구 및 알림 유형 ⓘ | 주파수 ⓘ | 구성원 권한**(우상단 people 아이콘 1개).
- **그룹 행(전개 chevron, 3종)**:
  - **필요한 작업 알림**(주파수 "다양한"): 전개 시 **9개 도구 행** — 브리지·비용·양식·이슈·회의록·검토·RFI·일정 및 작업 계획·자료제출. 각 도구를 **다시 전개**하면 도구별 **이벤트 행**(라벨+설명문 2줄, 원본 한/영 혼재 — 예: 양식 → "Form assigned to you"/"Comment mentioned"/"섹션 지정됨"/"상태가 변경됨"). **3단 계층**(그룹→도구→이벤트).
  - **기타 알림**(주파수 "다양한"): 전개 시 **15개 도구 행** — 브리지·서신·비용·Design Collaboration·파일·양식·이슈·회의록·AutoSpecs·검토·RFI·일정 및 작업 계획·시트·자료제출·자료전송.
  - **Upcoming and overdue item reminders**(주파수 "매일").
- 각 도구/이벤트 행 컨트롤: **주파수 드롭다운**(네이티브 `<select>`, 옵션값 고정 4종: **즉시·매시·다양한·매일**, 선택 시 로컬 state만 변경) + **권한 토글바**(정적 3세그먼트 막대 + "관리" 라벨, 클릭 비활성 — affordance 표시만).
- **이벤트 라벨 정책**: 캡처 문자열(한/영 혼재) 그대로 재현. 미상값은 자리표시자 허용.
