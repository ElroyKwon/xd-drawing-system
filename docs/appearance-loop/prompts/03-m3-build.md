# M3 메타프롬프트 — Build 비뷰어 표면 (executor contract)

> ai-loop Phase 2.5 산출물. **freeze v1**(2026-06-24, 4개 핵심 결정 AskUserQuestion 공동설계 확정). 구현 에이전트는 이 지시에 맞춰 실행하고, 별도 검증팀은 아래 **Acceptance checklist**를 항목별 채점한다. 기준 변경 = 스코프 변경 = HUMAN_GATE.

## Stage goal

기존 `src/BuildSheetsView.tsx`(754줄 단일 파일)의 Build 비뷰어 표면 중 **캡처 근거가 강한 3개 화면(홈·시트·파일)**을 카탈로그(§C·§D·§I) 구조 수준으로 끌어올리고, 화면별 컴포넌트를 별도 파일로 분할한다. "없는 화면 만들기"가 아니라 캡처가 기록한 영역·컨트롤·탭·컬럼·빈상태의 갭 메우기. 2D 뷰어/마크업(`SheetViewerShell`)은 M4 대상 — 본 마일스톤에서 내용 변경 없음(분할 이동만).

## Co-design log (frozen decisions — 2026-06-24, AskUserQuestion)

1. **범위 = 캡처 강한 3개 집중 + 나머지 6개 이월.** 홈(개요/종합 탭)·시트(행메뉴 보강)·파일(폴더트리·Welcome 배너·11컬럼·업로드 모달)만 카탈로그 수준으로. **이슈·양식·사진·구성원·브리지·설정은 Build 레벨 독립 캡처가 없으므로 현 셸/placeholder 그대로 유지**(M2 "캡처 없으면 추정 구현 금지" 원칙 일관). → LOOP Done-When FR-FS-005/006/011은 본 M3가 커버, **FR-FS-012(이슈)/013(양식)/014(사진)/015(구성원·브리지·설정)는 캡처 부재로 후속 이월**(PROGRESS Done-When 이월 추적에 기록 — Phase 6.5 reconcile에서 UNMET 오판 방지).
2. **상호작용 깊이 = M1·M2 계승(구조 우선).** 살아있는 로컬 state: 홈 개요/종합 탭 전환, 파일 폴더트리 선택, 파일 업로드 모달 열고닫기, 시트 행메뉴 케밥 팝오버 열고닫기. 정적 affordance(영속/실연동 없음): 파일 업로드 실행, 검색·필터, 페이저, 내보내기/공유 액션, 진행률 실계산, 날씨 연동.
3. **종합 탭 분석 차트 = 빈상태 골격 + 정적 축/범례.** 6개 카드 골격 + "표시할 데이터 없음" 안내 + 막대차트 2종("작성 날짜별 이슈 상태"·"매일 완료하는 양식")은 **정적 축/범례만**(캡처 그대로 재현). 실제 SVG 데이터 차트 렌더는 과잉 — 안 함.
4. **코드 구조 = 화면별 파일 분할.** 각 뷰 컴포넌트를 개별 파일로 추출(`src/build/` 하위). `BuildSheetsView.tsx`는 rail+topbar+라우팅 셸만 남긴다. 손대지 않는 6개 화면도 같은 분할의 일관성을 위해 추출하되 **로직·마크업 무변경(순수 이동)** → 기존 테스트 PASS가 분할 무회귀 증거.

### 화면 1 — Build 홈 (캡처 174415·174446·174503·175056·175107) [보강]
현 `BuildHomeView`는 4카드 셸·"68%" 하드코딩·탭 없음. 카탈로그 수준으로:
- **개요/종합 탭** 추가(`role="tablist"`, 로컬 state 전환). 기본 "개요".
- **개요 탭** 본문:
  - 인사 헤더("OO 님, 환영합니다." + 일러스트 자리) — 캡처 톤. (간단한 일러스트 placeholder 허용.)
  - **프로젝트 진행률** 카드: "% 완료 · 남은 일수 · 목표 완료일" 표시(정적 mock 값) + 온보딩 박스("프로젝트 일정을 추가하거나…" 안내문). (빈상태 변형 "타임라인 미구성 + 날짜 추가"는 선택 — 한 가지 상태면 충분.)
  - **빠른 링크** 카드: 시트 N · 구성원 N + [시트][구성원] 버튼.
  - **현장 날씨** 카드: "대체로 흐림" + 온도/습도/풍속 + 더보기(Weather). (정적.)
  - **작업 상태** 카드: "나에게 할당된 작업" 행 + "프로젝트에 할당된 이슈" 행 + `>` 화살표.
  - **브리지** 카드(빈상태).
  - 우측 **최근 작업** 카드: "모두 보기" + 타임라인 항목 1~2건.
- **종합 탭** 본문(결정 3): 6개 분석 카드 2열 그리드, 각 우상단 필터/팝오버 아이콘:
  1. 이슈를 완료하는 데 걸리는 평균 시간 — "표시할 이슈 데이터 없음" 빈상태.
  2. 표시할 기한이 지난 이슈 — 빈상태.
  3. 작성 날짜별 이슈 상태 — **정적 축(수/날짜) + 범례(진행 중·완료·답변됨·보류·거부)**.
  4. 양식을 완료하는 데 걸리는 평균 시간 — "표시할 양식 데이터가 없습니다." 빈상태.
  5. 표시할 기한이 지난 양식 — 빈상태.
  6. 매일 완료하는 양식 — **정적 축 막대차트 골격**.
