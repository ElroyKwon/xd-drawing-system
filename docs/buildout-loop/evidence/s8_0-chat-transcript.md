# S8.0 사이드카 — 실 데이터 채팅 기록 (2026-07-03, 세션14)

> **무엇이 진짜인가**: 아래 모든 툴 호출과 JSON 결과는 **실 구동**이다.
> 8000 백엔드는 `store_backend=typedb`(Docker typedb/typedb:3.7.3, healthy)로 기동,
> 8001 사이드카가 8000 공개 HTTP API로 그라운딩 데이터를 읽었다. 프로젝트=`Study_Project`.
>
> **아직 진짜가 아닌 것**: LLM은 미배선(S8.1/S8.4). Ollama 미기동·API 키 미설정 상태라
> 사이드카가 스스로 자연어를 생성하지 못한다. 그래서 **자연어 답변은 에이전트가 실 툴 출력에서
> 구성**했다 — LLM이 붙으면 동일 툴·동일 데이터로 이 답을 자동 생성하게 된다. 즉 이 기록은
> **데이터 경로의 사람이 읽는 프리뷰**이며, 답에 인용된 수치·ID는 전부 실측이다.
>
> 사이드카 health(실측): `{"ok":true,"backend_8000":{"reachable":true,"current_user":"member-owner"}}`

**tool** `list_sheets(project="Study_Project", discipline="E")` → 실 결과 `count=1`

**답** (툴 결과 기반):
Study_Project에는 완료 처리된 시트가 **총 15장** 있습니다. 공종 분류상 **전기(E) 1장**은

있습니다 — 여기엔 배치도(plan_a/plan_b/plan_v1)와 `제주_BESS_전기도면_8p`(Page 1~8)가 포함됩니다.

## 정리 — 이 기록이 증명하는 것

- **그라운딩 데이터 경로가 실제로 동작**: 8001 → 8000(TypeDB) HTTP → `search`/`list_sheets`가
  실 시트/이슈/파일을 반환. 인용된 15시트·케이블 3이슈·BESS 8p·접지 이슈는 전부 실측.
- **딥링크 ID가 안정적으로 흐름**: 답마다 sheet_id/issue_id를 실어 프론트 챗 드로어(S8.3)에서
  "이 시트로 이동" 브리지가 가능함을 보임.
- **남은 것 = LLM 배선(S8.1 provider + 오케스트레이션, S8.4 egress)**: 위 자연어 답을 사이드카가
  스스로 생성하려면 provider가 필요. 현재 로컬(Ollama) 미기동·클라우드 키 미설정.
  → 로컬 모델 기동(egress 0, 추천) 또는 API 키+egress 게이트 결정 필요.
