# S2.5 — 멀티페이지 스케일 강건화 (메타 추출 + 목록 스케일 + 용량 가시화)  [STATUS: FROZEN — 2026-06-29 사용자 확인 완료]

> ai-loop 스테이지 계약. `LOOP.md`·`PLAN.md` freeze 결정과 S1~S3 결과를 상속한다. **2026-06-29 68p/94MB 전기도면 PDF 실측**(EVIDENCE S2.5 섹션)으로 도출된 스케일 부채를 해소한다. 구현 에이전트가 이 텍스트를 그대로 입력받아 자율 실행하고, **별도 검증팀이 아래 Acceptance checklist(F1~F10)로 항목별 채점**한다. 합격을 위해 기준을 도중에 고치지 않는다(기준 변경 = 스코프 변경 = HUMAN_GATE).

## 배경 — 실측으로 확정된 부채

68페이지/94MB 전기도면 PDF(제주 BESS, 한화 양식)를 end-to-end로 통과시킨 결과:
- 변환 ~15초·PNG 68/68·API 31KB/<100ms = **성능/용량/변환속도는 견딤**(병목 아님).
- **시트번호·제목 추출 0% → 100% "Page N" 폴백**(공종 100% 'G'). PDF에 텍스트 레이어가 멀쩡히 있는데도(2p=2967자, `DWG NO`·`DWG. TITLE` 라벨 명확) 추출 실패 = **휴리스틱 양식 종속**이 진짜 병목.
- 목록 페이지네이션 버튼은 **장식**("1 중 1" 하드코딩, onClick 없음). 전체 `.map()` 일괄 렌더, 가상화 없음.
- 도면당 **119MB**(원본 PDF 93MB + PNG 27MB) 저장.

근본 원인(코드 확인): `sheet_meta.py` `_filename_number`가 한글 파일명에 None → 폴백 불가 / `_near_label_number`가 라벨 다음 2줄만 보는데 PDF 2D 타이틀블록은 `get_text()` 선형화에서 라벨↔값이 인접하지 않음 / `_title`이 파일명 stem 우선이라 멀티페이지는 전 시트가 동일 제목.

## Stage goal / Done-When

전기도면처럼 **수백 장 규모 멀티페이지 PDF**가 와도 시트 목록이 사람이 읽을 수 있게 뜨고, 수백 행에서도 목록이 버티고, 저장 용량이 가시화된다.

**완료 정의**: 제주 BESS 68p PDF를 다시 올리면 → 시트번호/제목/공종이 폴백이 아닌 **실제 추출값**으로 유의미하게 채워지고(타이틀블록에 텍스트가 실재하는 페이지 한정) → 시트 목록이 페이지당 N개로 **실제 페이지네이션**되며 공종 필터·정렬과 결합하고 → 도면별 저장 용량이 UI에 표시된다. 청주 단일파일 경로(S2)는 회귀 없음. 콘솔 0.

## Co-design log (2026-06-29 사용자 확정 — freeze 대상)

- **(P1) 메타 추출 = 라벨 앵커 + 폴백.** 텍스트 레이어에서 `DWG NO`/`DWG. TITLE`/`도면번호`/`도면명` 라벨을 찾아 **인접/우측/하단 값**을 추출하되, `get_text()` 선형 순서에 의존하지 않도록 **PyMuPDF의 좌표 기반 텍스트 블록(`get_text("words")` 또는 `"dict"`)으로 라벨-값을 공간적으로 페어링**한다(라벨 토큰의 오른쪽·아래 근접 토큰). 도면번호 정규식을 한화 양식까지 넓힌다(현 `_CAND_RE`가 못 잡는 형식 보강). 실패 시 기존 폴백 체인(파일명 번호 → "Page N") 유지. 한글 파일명도 내부 토큰에서 번호 시도.
- **(P2) 목록 스케일 = 클라이언트 페이지네이션.** 현재 장식뿐인 페이저를 실동작화(페이지당 50, 이전/다음·현재/총, 공종 필터·검색·정렬 결합, 필터 변경 시 1페이지 리셋). 신규 라이브러리 도입 없음(의존성 0). 윈도 가상화/서버 페이지네이션은 후속.
- **(P3) 용량 = 현행 보관 유지 + 가시화(추천안).** 원본 PDF 보관 정책은 현행 유지(삭제/압축/외부참조 안 함). 대신 **도면별 저장 용량(원본+파생)을 백엔드 stat으로 노출**하고 파일/시트 UI에 표시해 운영자가 인지하게 한다. 압축·보존정책·외부 스토리지는 명시적 후속 부채.
- **(가정) 측정 기준.** P1 성공은 "제주 68p에서 번호/제목/공종 폴백률이 실측 대비 유의미하게 하락"으로 판정하되, **타이틀블록에 추출할 텍스트가 실재하는 페이지에 한함**(빈 표지·인덱스 페이지는 폴백이 정상). 무성 100% 목표를 강요하지 않음 — 처리 결과를 EVIDENCE에 정직히 기록(무성 절단 금지). 청주 40개 개별 파일 경로는 회귀 0.
- **(가정) 범위 외.** OCR(스캔 PDF 텍스트화), AI/LLM 추출, 양식 프로파일 등록 UI, 윈도 가상화, 서버 페이지네이션, 원본 압축/보존정책 자동화는 S2.5 범위 밖.

## Instruction (수행 단계)

1. **S2.5-a 라벨 앵커 추출**: `sheet_meta.py`에 좌표 기반 라벨-값 페어링 경로 추가. `conversion.py`(PDF 분할부)가 페이지 텍스트뿐 아니라 **좌표 포함 토큰**(`page.get_text("words")` 등)을 `extract_sheet_meta`에 넘기도록 시그니처 확장(기존 텍스트 경로는 폴백으로 유지, S2 호환). 라벨 토큰의 우측/하단 근접 토큰을 값 후보로. 도면번호 정규식 보강.
2. **S2.5-b 멀티페이지 제목**: `_title`이 멀티페이지 PDF에서 페이지별 타이틀블록 도면명을 우선하도록(파일명 stem은 단일파일 폴백으로 격하). 전 시트 동일 제목 방지.
3. **S2.5-c 용량 stat**: 백엔드가 도면별 저장 용량(원본+sheets PNG+벡터)을 계산해 목록/단건 API 응답에 `storage_bytes`(또는 동등 필드)로 노출.
4. **S2.5-d 클라 페이지네이션**: `SheetsListView.tsx` 페이저 실동작화(페이지당 50, 상태 hook, 이전/다음 비활성 경계, "X 중 Y", 공종 필터·검색·정렬과 결합·필터 변경 시 리셋). 카운트 라벨 실데이터.
5. **S2.5-e 용량 표시**: 파일/시트 UI에 도면별 용량 표시(`FilesView` 또는 시트 목록 적절한 위치). 과도 설계 금지 — 숫자 표시 수준.
6. **검증**: 백엔드 pytest(라벨앵커 추출·좌표 페어링·정규식 보강·멀티페이지 제목·용량 stat·청주 회귀), 프론트 test(페이지네이션 동작·필터 결합·경계·용량 표시), `npm run build`·`npm test`·`git diff --check`. 브라우저 e2e(제주 68p 재업로드 → 번호/제목/공종 실값 → 페이지네이션 넘김 → 용량 표시) 스크린샷, 콘솔 0. EVIDENCE에 추출 폴백률 before/after 수치 기록.
