## Stage goal / Done-When

ACC 벤치마크 **§C Build 프로젝트 홈**과 **전역 검색**을 **실데이터+실동작**으로 만든다. 현재 `BuildHomeView`는 모든 수치가 인라인 하드코딩(진행률 1%·구성원 1·작업 1개·이슈 1개·날씨·최근활동·분석 6카드)이고, 시트 수 prop조차 홈에선 `listDrawings`를 호출하지 않아 항상 0이다. Build 워크스페이스에는 전역 검색 입력 자체가 없고 백엔드 검색 라우트도 없다. LOOP.md Done-When "**Build 홈 위젯 실데이터 + 전역 검색**"을 충족한다.

두 축:
1. **홈 위젯 실데이터** — 집계 가능한 위젯(시트 수·파일/폴더 수·이슈 상태별 수·최근 업로드·저장 용량·작성일별 이슈 상태 차트)을 실 API 집계로 교체. 가짜 하드코딩 수치 제거.
2. **전역 검색** — 백엔드 `GET /api/search` 신규(시트·이슈·파일·폴더 교차) + Build 상단바 전역 검색 입력 + 결과 패널 + 결과 클릭 시 해당 시트/이슈/파일로 딥링크(S5 딥링크 패턴 계승).

**완료 정의**: (a) 홈 진입 시 실제 프로젝트의 시트/파일/폴더/이슈 수, 최근 업로드, 저장 용량이 실집계로 표시되고 이슈 탭·시트 목록의 실제 수와 정확히 일치한다. (b) 작성일별 이슈 상태 분석 차트가 실 이슈 데이터로 그려진다(신규 의존성 0, 경량 인라인 SVG/CSS). (c) 백엔드 없는 위젯(구성원·작업·브리지·양식·날씨·일정 진행률)은 가짜 수치를 제거하고 ACC 빈 신규 프로젝트와 동일한 정직한 빈 상태로 표시한다. (d) 상단 전역 검색에 질의하면 시트·이슈·파일/폴더를 가로질러 결과가 타입별로 표시되고, 클릭하면 해당 대상으로 뷰어/탭이 점프한다. 콘솔 0.

## Co-design log (2026-06-30 사용자 확정 — AskUserQuestion 4결정 freeze)

- **(Q1) 미구현 위젯 = 정직한 빈 상태 유지.** 백엔드 데이터원이 없는 위젯(구성원·작업[task]·브리지·양식[forms]·현장 날씨·일정 기반 진행률)은 가짜 하드코딩 수치(1%·구성원 1·작업 1개·날씨 9℃ 등)를 **제거**하고, ACC 빈 신규 프로젝트와 동일하게 "작업 없음"·"콘텐츠 없음"·"데이터 없음"·"추가 예정" 빈 상태로 표시. 외관(카드 레이아웃)은 보존. → 구성원=S7, 작업/브리지/양식 백엔드=미계획 후속, 날씨=외부 API 범위 외.
- **(Q2) 집계 범위 = 카운트 + 이슈 분석 차트.** 개요 위젯의 실집계(시트 수·파일/폴더 수·이슈 상태별 수·최근 업로드·저장 용량) + 종합(Analytics)의 "작성일별 이슈 상태" 차트를 이슈 실데이터로. **차트는 신규 의존성 0**(경량 인라인 SVG/CSS 막대·누적). 양식(forms) 기반 분석 카드는 백엔드 없으므로 빈 상태. 이슈 "기한 지남"은 due_date 모델 부재로 범위 외.
- **(Q3) 전역 검색 = 백엔드 라우트 + 상단 전역검색.** 신규 `GET /api/search?q=&project_name=` (서버측 교차 검색) + Build 상단바 전역 검색 입력 + 결과 패널. 프론트 전용 통합 검색(클라 합산)이 아니라 서버측 검색이 정석. 결과 클릭 → 딥링크.
- **(Q4) 검색 대상 = 시트 + 이슈 + 파일/폴더 전부.** 프로젝트 내 모든 객체를 가로질러 검색: 시트 번호/제목, 도면 파일명, 이슈 제목, 폴더명. 대소문자 무시 부분일치.
- **(가정) 영속/집계 = 기존 store·API 재사용.** 신규 엔티티 없음 — `list_drawings`/`list_issues`/`list_folders` 등 기존 집계를 조합. 검색도 기존 store 목록을 서버측에서 필터. TypeDB 직접 쿼리화는 후속(JSON 미러 유지).
- **(가정) 홈 로드 버그 수정.** `BuildSheetsView` 폴링이 `activeSection`이 시트/이슈일 때만 `listDrawings`를 호출해 홈에선 시트 수가 0 — 홈도 실데이터를 로드하도록 수정(또는 BuildHomeView가 자체 집계 fetch).
- **(가정) 딥링크.** 검색 결과 → 시트는 뷰어 진입(S2 openSheet 패턴), 이슈는 이슈 탭+선택(S5 focusIssue/openIssuePin 패턴), 파일/폴더는 파일 화면+폴더 선택. 재사용.

## Instruction (수행 단계)

1. **S6-a 백엔드 검색 라우트**: 신규 `backend/routes_search.py` — `GET /api/search?q=&project_name=` → 시트(완료 도면의 sheets: 번호/제목)·이슈(title)·파일(도면 filename)·폴더(name)를 대소문자 무시 부분일치로 교차 검색. 응답은 타입별 그룹 + 딥링크 식별자(`file_id`/`sheet_id`/`issue_id`/`folder_id`) + 표시 라벨. 빈/공백 `q` → 빈 결과(또는 400 일관). 결과 수 상한(타입별 상한, 초과 시 `truncated` 표기 — 무성 절단 금지). `main.py` 라우터 등록.
2. **S6-b 프론트 검색 API 클라이언트**: `src/api/drawings.ts`에 `SearchResult` 타입 + `searchProject(projectName, q)` 추가(S5 API 패턴 계승).
3. **S6-c 홈 집계 데이터 로드**: `BuildHomeView`(또는 호출부 `BuildSheetsView`)가 홈 진입 시 `listDrawings`·`listIssues`·`listFolders`를 실제 호출해 집계. 홈에서 시트 수가 항상 0인 버그 수정. 집계 헬퍼는 순수 함수로 분리(테스트 가능).
4. **S6-d 홈 위젯 실데이터 교체**: `BuildHomeView` 인라인 하드코딩 제거 — 빠른 링크 시트 수(완료 시트)·파일 수·폴더 수, 작업 상태/개요의 이슈 수(status 실집계), 최근 활동(최신 업로드 N건, `upload_date`), 저장 용량(`storage_bytes` 합)을 실데이터로. 수치는 다른 화면과 정합.
5. **S6-e 분석 차트 실데이터**: 종합(Analytics)의 "작성일별 이슈 상태" 차트를 `listIssues` 작성일×상태 집계로 그린다 — **경량 인라인 SVG/CSS**(신규 의존성 0). 더미 1~6월 틱·정적 `analyticsCards` 제거. 양식 기반 카드는 빈 상태.
6. **S6-f 미구현 위젯 빈 상태**: 구성원·작업·브리지·양식·날씨·일정 진행률 위젯의 가짜 하드코딩을 정직한 빈 상태로 교체("작업 없음"·"콘텐츠 없음"·"데이터 없음"·"추가 예정"). 외관 카드 보존, 가짜 수치 0건.
7. **S6-g 상단 전역 검색 UI**: Build 상단바(`BuildSheetsView` build-topbar)에 전역 검색 입력 신규. 입력(디바운스) → `searchProject` → 결과 패널(타입별 그룹: 시트/이슈/파일/폴더). 빈 쿼리·결과 없음·로딩 상태 처리. a11y(라벨·role).
8. **S6-h 검색 결과 딥링크**: 결과 항목 클릭 → 시트=뷰어 진입(openSheet), 이슈=이슈 탭+focusIssue/딥링크, 파일/폴더=파일 화면+폴더 선택. S2/S5 배선 재사용.
9. **검증**: 백엔드 pytest(검색 라우트: 타입별 교차·부분일치·빈 쿼리·상한/truncated·project 스코프·에러계약, 집계 정합), 프론트 test(홈 집계 헬퍼·위젯 실데이터·빈 상태·검색 결과 렌더·딥링크 콜백), `npm run build`·`npm test`·`git diff --check`. 브라우저 e2e(홈 실데이터 위젯·차트, 전역 검색 시트/이슈/파일 교차→딥링크) 스크린샷, 콘솔 0.
