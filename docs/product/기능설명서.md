# XD 도면관리 시스템 — 기능설명서

> ACC(Autodesk Construction Cloud) Build 화면을 벤치마크로, 건설·플랜트 전기도면 관리를 위해 개발한 웹 기반 도면관리 시스템의 구현 기능 명세.
> 작성: 2026-07-02 · 대상 버전: 빌드아웃 S1~S9.1 (실동작 전 기능)

---

## 1. 개요

XD 도면관리 시스템은 DWG/DXF/PDF 도면을 업로드하면 **서버에서 자동 변환·시트 추출**하고, 웹 브라우저에서 **실제 도면을 렌더링·마크업·측정·비교**하며, **이슈·작업·양식**으로 현장 협업을 관리하는 로컬 풀스택 애플리케이션이다.

- **프런트엔드**: React + TypeScript + Vite (정적 산출물, 브라우저 SPA)
- **백엔드**: FastAPI(Python) 로컬 서버 — 도면 변환(ODA File Converter / ezdxf / PyMuPDF), 메타 영속(JSON 스토어 + TypeDB 옵션)
- **저장소 추상화**: `DrawingStore` — JSON 파일 스토어를 기본, TypeDB 온톨로지 백엔드를 옵션으로 지원(런타임 비의존)
- **권한 모델**: 로컬 모의 인증 + 역할 기반 접근제어(관리자 / 편집자 / 뷰어)

전 기능은 **정적 목업이 아니라 실데이터·실연산·영속**으로 동작한다.

## 3. 모듈별 기능

### 3.1 도면 업로드 · 자동 변환 · 2D 뷰어

- **업로드**: DWG / DXF / PDF 파일을 드래그앤드롭 또는 선택 업로드. 서버가 비동기 변환하고 진행 상태를 폴링 표시.
- **변환 파이프라인**: DWG → (ODA File Converter) → DXF → (ezdxf) 벡터 추출 / PDF → (PyMuPDF) 페이지 분할·래스터.
- **2D 벡터 뷰어**: DXF에서 추출한 벡터 엔티티(LINE·LWPOLYLINE·ARC·CIRCLE·TEXT·MTEXT·INSERT·HATCH·DIMENSION 등)를 canvas2D로 **무손실 줌·팬·핏**, 레이어 토글. PDF는 고해상도 래스터 이미지로 렌더.
- **렌더 방식 결정**: 하이브리드 래스터 vs 오픈소스 벡터 2-way bake-off 후 **벡터(canvas2D) 채택** — 무손실 확대와 레이어 제어, 벡터 좌표 기반 인터랙션이 가능. (Autodesk APS는 벤더 비종속 전략상 배제.)

### 3.2 시트 레지스터

- 업로드 도면에서 **시트를 자동 추출**해 목록으로 등록(정적 시드 없음, 100% 실데이터).
- PDF는 페이지별로 분할, DWG는 모델스페이스/페이퍼스페이스 분리.
- **타이틀블록 휴리스틱**으로 시트번호·제목·공종(A 건축 / E 전기 / M 기계 / P 배관 / G 기타)을 추출.
- 공종 필터, 시트번호 자연 정렬, 검색, 그리드/리스트 보기, 하단 썸네일 필름스트립.

### 3.3 마크업 · 측정 · 시트 비교

- **마크업**: 텍스트 · 도형 · 리비전 클라우드 · 폴리라인 · 다각형 · 펜. 색상·굵기·불투명도 속성. (`05` 참조 — 단선결선도에 리비전 클라우드+주석)
- **좌표 이중 트랙**: DXF 벡터는 world(모델) 좌표, PDF/래스터는 정규화 이미지 좌표[0,1]로 저장 — 줌 배율과 무관하게 정확한 위치 고정.
- **측정**: DXF의 도면 단위(`$INSUNITS`)를 읽어 **실척 환산** — 선형 거리, 다각형 면적, 지름 측정.
- **시트 비교**: 같은 버전세트의 두 버전을 클라이언트 색상 오버레이(이전=빨강 / 현재=파랑) + 백엔드 픽셀 diff로 변경 영역 시각화.
- 모든 마크업/측정은 (파일, 시트) 스코프로 **영속**되어 새로고침·재접속 후 복원.

### 3.4 파일 · 폴더 · 버전 · 권한

- ACC식 기본 폴더 트리(Bids · Contractors · Coordination · Drawings · For the Field · Handover 등) 자동 시드 + 폴더 CRUD.
- **명시적 버전세트**: 같은 도면의 새 버전을 추가하면 이력 보관, 목록엔 최신 1건 표시.
- 파일 다운로드, 삭제, 폴더 타깃 업로드, 폴더 공유 상태 편집.

### 3.5 이슈 관리 (`07`)

- **독립 이슈 엔티티** — ACC식 상태 머신(열림 → 진행중 → 답변됨 → 닫힘, 삭제됨은 소프트 삭제).
- 유형(설계 검토·현장 확인·간섭·품질·협의), 카테고리(clash·quality·coordination), 담당자, 설명.
- **뷰어 핀 연계**: 이슈를 도면 좌표에 핀으로 고정(선택적). 목록 ↔ 뷰어 **양방향 딥링크** — 이슈에서 도면 위 핀 위치로 점프.
- 좌표계는 마크업과 동일(DXF world / PDF image). 핀 없는 프로젝트 전역 이슈도 허용.

### 3.6 작업(Tasks) (`08`)

- 프로젝트 전역 작업 항목 — 제목·담당자·**상태(할 일 / 진행중 / 완료)**·**우선순위(높음 🔴 / 보통 🟡 / 낮음 ⚪)**·기한.
- 상태 필터, 검색, 작성·상태변경·삭제. 미완료 우선 정렬, 완료 항목은 취소선.
- 홈 대시보드 `작업 상태` 위젯에 진행/완료 카운트 실시간 집계 + 딥링크.

### 3.7 양식(Forms) — 체크리스트 점검표 (`09`)

- **체크리스트 기반 점검표** — 유형(점검·안전·품질·검사)·상태(미시작·진행중·제출·완료)·담당자·기한.
- 항목별 체크박스 → **완료율 자동 산출**(체크된 항목 비율)과 진행바 표시.
- 상태 필터, 작성(항목 줄 단위 입력), 항목 토글, 상태변경, 삭제.
- 홈 종합 탭에 **양식 KPI 카드**(등록 건수 · 평균 완료율 · 미완료 건수) 실데이터 집계.

### 3.8 프로젝트 홈 대시보드 (`02`, `03`)

- **개요 탭**: 진행률, 빠른 링크(시트·파일·폴더 수), 저장 용량, 작업 상태 위젯, 최근 활동(실 업로드 이력).
- **종합 탭**: 작성일별 이슈 상태 누적 막대 차트 + 양식 분석 카드. 의존성 없는 인라인 SVG/CSS 차트.
- 미구현 위젯은 가짜 데이터 대신 **정직한 빈 상태**로 표시.

### 3.9 전역 검색

- 상단 검색바에서 **시트·이슈·파일·폴더를 가로질러** 부분일치 검색.
- 결과 클릭 시 해당 화면으로 딥링크(시트→뷰어, 이슈→이슈탭 선택, 폴더→파일 화면).

### 3.10 인증 · 권한(RBAC) (`10`)

- 로컬 모의 사용자 전환(비밀번호 없이 구성원 전환) + 프로젝트/구성원 영속.
- **역할 기반 접근제어**: 관리자 > 편집자 > 뷰어.
  - 뷰어는 업로드·폴더·마크업·측정·이슈·작업·양식의 작성/수정/삭제가 **UI에서 비활성화**되고 서버에서도 403으로 차단(이중 방어).
- 프로젝트 생성자는 자동으로 관리자 등록. 마지막 관리자 강등/제거 방지.

## 4. 데모 데이터(전기 도면 기준)

문서·시연용으로 전기 도면(6.6kV 변전설비 단선결선도, 제주 BESS 전기도면 8p, 배치 DWG)에 현실적 데이터를 시딩한다(재현 스크립트 `scripts/seed_demo.py`):

- **이슈 12건**(핀 11건): 차단기 정격 확인, 인입 케이블 규격(CV 400sq), 접지선 굵기, 케이블 트레이 간섭, 수전실 이격거리, ATS 용량, 조명 분전반 회로 증설, 피뢰 인하도선, PCS 환기 등. 각 핀 위치에 리비전 클라우드 + 실무 주석 마크업.
- **작업 8건**: 접지저항 측정 제출, 차단기 정격 승인, 인입 케이블 발주, 트레이 시공 상세도, 발전기 시운전 계획 등(상태·우선순위·기한 분산).
- **양식 5건**: 수배전반 인수 점검표, 접지 시스템 시공 점검표, 전기실 소방·안전 점검표, 케이블 포설 품질 점검표, 비상발전기 시운전 점검표(완료율 0~100%).

## 5. 품질 · 검증

- 프런트엔드 단위 테스트 **108건**, 백엔드 테스트 **86건** 통과.
- 각 기능은 브라우저 end-to-end로 실동작 검증(콘솔 에러 0).
- 독립 검증 3렌즈(적대적 백엔드·비기능 프런트·완료조건 비평)로 핵심 스테이지 결함 적발·수리.
