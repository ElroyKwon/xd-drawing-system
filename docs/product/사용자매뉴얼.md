# XD 도면관리 시스템 — 사용자 매뉴얼

> 실무 사용자를 위한 작업별 안내서. 화면 예시는 `docs/product/screenshots/` 참조.
> 작성: 2026-07-02

---

## 1. 시작하기

### 1.1 시스템 실행 (로컬)

1. **백엔드 서버 실행**
   ```
   cd backend
   XD_STORE=auto .venv/Scripts/python.exe -m uvicorn main:app --host 127.0.0.1 --port 8000
   ```
   - `XD_STORE=auto`: TypeDB가 있으면 사용, 없으면 JSON 스토어로 폴백.
2. **프런트엔드 실행**
   ```
   npm run dev
   ```
   브라우저에서 `http://127.0.0.1:5173` 접속.

### 1.2 로그인 · 사용자 전환

- 우측 상단 **사용자 메뉴**에서 현재 사용자를 전환한다(로컬 모의 인증).
- 사용자 역할에 따라 편집 가능 범위가 달라진다:
  - **관리자**: 모든 편집 + 구성원·권한 관리
  - **편집자**: 도면·마크업·이슈·작업·양식 편집
  - **뷰어**: 조회만 가능(편집 버튼 비활성화)

---

## 2. 프로젝트 (Hub)

1. Hub의 **프로젝트 목록**에서 프로젝트를 확인한다. (`01-hub-projects.png`)
2. `+ 프로젝트 만들기`로 새 프로젝트를 생성한다. 생성자는 자동으로 관리자가 된다.
3. 프로젝트 행의 **`Build 열기`**로 작업공간에 진입한다.

---

## 3. 도면 업로드 · 열람

### 3.1 업로드
1. Build 좌측 메뉴 **파일** → `파일 업로드`. (`06-files.png`)
2. 폴더를 선택하고 DWG / DXF / PDF를 올린다.
3. 변환이 완료되면 **시트** 목록에 자동 등록된다(진행 상태 표시).

### 3.2 도면 열람
1. **시트** 메뉴에서 도면을 찾는다. 공종 필터·검색·정렬 사용. (`04-sheets.png`)
2. 시트 행의 `열기`로 **2D 뷰어**를 연다. (`05-viewer-markup.png`)
3. 마우스 휠로 줌, 드래그로 팬, 하단 컨트롤로 화면 맞춤(Fit).
4. DXF 도면은 우측에서 **레이어를 토글**할 수 있다.

---

## 4. 마크업 · 측정

### 4.1 마크업 그리기
1. 뷰어 우측 도구 모음에서 도구 선택: **텍스트 · 도형 · 클라우드 · 폴리라인 · 다각형 · 펜**.
2. 도면 위에 그린다. 색상·굵기·불투명도를 속성에서 조정.
3. 좌측 **마크업 로그** 탭에서 작성 이력을 확인한다.
4. 마크업은 자동 저장되어 재접속 후에도 유지된다.

### 4.2 측정 (DXF)
1. 우측 **측정** 도구 선택 → 측정 종류(선형 / 면적 / 지름).
2. 도면 위 지점을 클릭하면 도면 단위 기반 **실척 값**이 표시된다.

### 4.3 시트 비교
1. 뷰어 상단 `시트 비교`.
2. 같은 도면의 두 버전을 선택하면 변경 영역이 빨강(이전)/파랑(현재)으로 표시된다.

---

## 5. 이슈 관리

### 5.1 이슈 작성
1. **이슈** 메뉴 → `이슈 작성`. (`07-issues.png`)
2. 제목·유형·카테고리·담당자·설명 입력 후 `작성`.

### 5.2 뷰어에서 핀 이슈 만들기
1. 뷰어 우측 **이슈 핀** 도구 선택 → 도면 위 문제 위치 클릭.
2. 이슈 정보를 입력하면 해당 좌표에 핀이 고정된다.

### 5.3 이슈 ↔ 도면 점프
- 이슈 상세의 `뷰어에서 핀 보기`로 도면의 핀 위치로 이동.
- 상태 드롭다운으로 진행 상태를 변경(열림→진행중→답변됨→닫힘).

---

## 6. 작업(Tasks)

1. **작업** 메뉴 → `작업 작성`. (`08-tasks.png`)
2. 제목·담당자·**우선순위**·상태·기한 입력.
3. 목록 상단 필터(할 일 / 진행중 / 완료)로 걸러본다.
4. 작업을 선택해 상태를 변경하거나 삭제한다. 홈 대시보드에서 진행/완료 현황을 확인.

---

## 7. 양식(Forms) — 점검 체크리스트

1. **양식** 메뉴 → `양식 작성`. (`09-forms-checklist.png`)
2. 제목·유형(점검/안전/품질/검사)·상태·담당자·기한 입력.
3. **점검 항목**을 한 줄에 하나씩 입력한다.
4. 상세에서 항목 체크박스를 켜면 **완료율 진행바**가 자동 갱신된다.
5. 검사가 끝나면 상태를 `완료`로 변경.

---

## 8. 홈 대시보드

- **개요 탭**: 진행률, 시트·파일·폴더 수, 저장 용량, 작업/이슈 현황, 최근 활동. (`02-build-home.png`)
- **종합 탭**: 작성일별 이슈 상태 차트, 양식 완료율 KPI. (`03-build-analytics.png`)
- 위젯의 숫자·항목을 클릭하면 해당 상세 화면으로 이동한다.

---

## 9. 전역 검색

- 상단 검색창에 키워드를 입력하면 **시트·이슈·파일·폴더**를 한 번에 검색.
- 결과를 클릭하면 해당 화면으로 바로 이동한다.

---

## 10. 구성원 · 권한

1. **구성원** 메뉴에서 프로젝트 구성원과 역할을 확인. (`10-members.png`)
2. 관리자는 구성원을 추가하고 역할(관리자/편집자/뷰어)을 변경할 수 있다.
3. 뷰어 계정으로 전환하면 모든 편집 버튼이 비활성화되는 것을 확인할 수 있다.

---

## 11. 자주 묻는 질문

**Q. 업로드했는데 시트에 안 보여요.**
A. 변환에 수 초~수십 초 걸립니다. 시트 화면은 자동 폴링하므로 잠시 기다리세요. DWG는 ODA File Converter 설치가 필요합니다.

**Q. 편집 버튼이 회색으로 비활성화되어 있어요.**
A. 현재 사용자가 **뷰어** 역할입니다. 편집자/관리자로 전환하세요.

**Q. 마크업이 도면 위 엉뚱한 곳에 있어요.**
A. 좌표계 문제일 수 있습니다. DXF는 world 좌표, PDF는 정규화 이미지 좌표로 저장되며 줌과 무관하게 위치가 고정됩니다.

**Q. 데모 데이터를 다시 채우려면?**
A. 백엔드 실행 상태에서 `PYTHONUTF8=1 backend/.venv/Scripts/python.exe scripts/seed_demo.py` (멱등 — 기존 데모 데이터를 초기화 후 재생성).
