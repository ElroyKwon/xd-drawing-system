## 기존 시나리오 유용성 평가 (요청 사항)

**"이 시나리오들이 실제 고객에게 도움이 되는가?"** 에 대한 평가입니다.

**결론: 유용하다. 다만 실데이터 그라운딩 + AI 가속을 더해야 "목업 냄새"가 사라진다.** — 그래서 이 세트로 보완했습니다.

| 항목 | 기존(A동 가상) | 본 세트(개정) |
|---|---|---|
| **친숙성** | ✅ 분전반 증설이라는 익숙한 전기공사로 앵커 — 강점 | 유지(380V 분전반 증설) |
| **신뢰성** | ⚠️ 가상 도면번호(E‑001)·태그(LP‑A31)·구 스크린샷 → 청주 엔지니어가 "실제와 다름"을 눈치챔 | ✅ **실 도면번호(EE‑01‑016)·실 설비(PP‑380V)·실 이슈("380V 분전반 회로 수 초과 — 증설 검토")·청주 실캡처** |
| **정직성** | ✅ 실증/구축/고도화 경계 명시 — 강점 | 유지·강화 |
| **차별화** | ⚠️ 전부 수동 루프 — 단순 뷰어와의 차이가 약함 | ✅ **AI 비전 문서로 "묻는 도면관리" 차별화**(실 gpt‑5.5 그라운딩) |
| **역할 흐름** | ✅ 운영자/CAD/문서관리자/협력사 — 강점 | 유지(실 구성원 3역할) |

**핵심 개선점 3가지**
1. **실데이터 그라운딩** — 가상 번호를 시스템에 실제 적재된 EE 도면·설비·이슈로 전면 교체. 고객이 "우리 도면이네"라고 인식.
2. **AI 가속 시나리오 신설** — 수동으로 네 화면 오가던 판단을 AI 한 질문으로. 실제로 동작함을 실험으로 입증(환각 검증 포함).
3. **정직한 포지셔닝 유지** — AI는 5단계 옵션(통합제안서 §9)과 어긋나지 않게 "이미 해 놨고 켜는 시점만 정한다"로 위치.

## 개발 메모 — 이 세트 제작 중 수리한 실사용 버그 2건

시나리오용 AI 실험 중 발견해 **제품 코드를 실제 수정**했습니다(테스트·빌드 통과).

- **AI 답변의 마크다운 표가 raw 파이프(`| a | b |`)로 노출** → 경량 렌더러(`src/ai/markdown.tsx`)에 **GFM 표·헤딩 렌더링 추가**. 이제 설비 표·증설 영향 표가 실제 표로 렌더됨(ai‑02/03). vitest +4.
- **드로어 위에서 마우스 휠 시 대화창 대신 뒤 페이지가 스크롤** → `.ai-messages`에 `min-height:0`(flex 스크롤 함정)·`overscroll-behavior:contain` 추가. 이제 드로어 위 휠은 대화창을 스크롤.
