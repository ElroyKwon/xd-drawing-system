# 설계 — 대화형 에이전트 액션 + MCP 로드맵

> 작성 2026-07-06. 공동설계(brainstorming) 산출물. 이 문서는 (A) 전체 로드맵(P1~P5) 컨텍스트와
> (B) 첫 서브프로젝트 **P1+P2(앱 내 대화형 액션)** 상세 설계를 담는다. 구현 계획은 writing-plans에서.

---

## A. 전체 로드맵 — "대화로 일하는 XD" + MCP 제공

**현재 상태:** AI 챗은 8001 사이드카에서 **읽기 전용 툴**(HTTP GET 9종)로 8000/TypeDB 데이터에
그라운딩하는 Q&A만 한다. 사이드카는 8000을 mutate하지 않는다(S8 격리 불변식: 8001→8000 write 0,
`src/ai`→앱 모듈 미의존).

**목표:** ① 사용자가 대화로 실제 write 액션(이슈·작업·업로드·이메일) 수행 ② 이 능력을 MCP로
노출해 외부 AI 클라이언트(Claude Desktop 등)·다른 사용자도 XD 구동.

### 분해 (의존 순서)

| # | 서브프로젝트 | 내용 | 게이트/선행 | 착수 |
|---|---|---|---|---|
| **P1** | 앱 내 대화형 액션 | 이슈 생성·상태변경·작업 생성을 AI가 **제안→확인→실행** | 없음 | **이번 설계** |
| **P2** | 안전·확인·감사 프레임 | 휴먼인더루프 확인 카드, RBAC, 액션 감사로그, 실패 격리 | P1과 함께 | **이번 설계** |
| P3 | MCP 서버(읽기) | XD read 툴을 Model Context Protocol로 노출 → 외부 클라이언트 조회 | 없음 | 후속 |
| P4 | 실 인증/세션 (GATE-6) | 로컬모의→실 자격증명·세션·API 토큰. 멀티유저 MCP write 전제 | **GATE-6** | 게이트 후 |
| P5 | MCP write + 이메일 egress (GATE-5) | 외부에서 write 액션·실 이메일 발송 | P4·**GATE-5** | 게이트 후 |

**설계 원칙:** "대화로 이슈 남겨줘/업로드해줘"의 핵심 가치(P1+P2)는 게이트 없이 지금 구현 가능.
멀티유저 MCP write(P4+P5)만 실인증·이메일 egress 게이트가 걸린다. P3(읽기 MCP)도 게이트 무관.

## B. P1+P2 상세 설계 — 앱 내 대화형 액션

### B.0 공동설계 확정(freeze)
- **액션 범위(첫 버전):** 이슈 생성 · 이슈 상태변경 · 작업 생성 (핵심 3종). 파일 업로드·이메일=범위 밖(P3~P5).
- **확인 모델:** **구조화 미리보기 카드 + 버튼 확인**. 클릭 전까지 서버 변경 0.
- **실행 위치:** **프론트 → 8000 직접**. AI(8001)는 제안만(부작용 0). 사이드카 격리 불변식 보존.

### B.1 아키텍처 & 데이터 흐름
```
사용자: "22.9kV 단선결선도에 차단기 정격 확인 이슈 남겨줘"
 → [8001] AI가 read 툴로 그라운딩(EE-01-001 sheet_id 확보)
 → [8001] propose_create_issue(...) 호출 = 부작용 0, 정규화된 액션 스펙만 반환
 → 챗 응답에 pending_action 실려 옴 (서버 변경 아직 0)
 → [프론트] ChatDrawer가 ActionCard 렌더(제목·상태·카테고리·대상시트)
 → 사용자 [실행] 클릭
 → [프론트] 기존 createIssue() → POST http://8000/api/issues (require_role 편집자)
 → 성공 → "✓ 이슈 생성됨 #123 [열기]" 시스템 버블 + 딥링크 칩(xd:navigate 재사용) + 감사로그
```

### B.2 컴포넌트 (신규/변경)

| 파일 | 종류 | 책임 |
|---|---|---|
| `backend/ai/actions.py` | 신규 | 액션 툴 3종 스키마 + 참조 해소(라벨→실 sheet_id/issue_id는 기존 read 경로). **8000 write 절대 호출 안 함**(제안 스펙만 생성·검증). |
| `backend/ai/agent.py` | 변경 | 액션 툴을 카탈로그에 등록. 호출 시 실행이 아니라 `pending_actions`로 수집. 시스템 프롬프트에 "추측 말고 제안·부족 필드 되물음·한 턴 한 액션" 지침. |
| `backend/ai/routes_chat.py` | 변경 | 챗 응답에 `pending_actions[]` 필드 추가. |
| `src/ai/ActionCard.tsx` | 신규 | 미리보기 카드 UI + [실행]/[취소]. 기존 `createIssue`/`patchIssue`/`createTask`(src/api) 호출. 성공 시 딥링크·시스템 버블, 실패 시 카드 에러. |
| `src/ai/ChatDrawer.tsx` | 변경 | `pending_actions` → ActionCard 렌더. `canEdit` 게이팅([실행] disabled). |
| `src/ai/aiClient.ts` | 변경 | `pending_actions` 타입. |
| `backend/`(8000) 액션 감사 | 신규(경량) | 실행이 프론트→8000이므로 **감사도 8000에** 둔다. 확인된 액션 write 라우트가 optional `origin="ai_chat"`·`conversation_id`를 받아 감사로그(actor·action_type·target_id·origin·시각)에 기록. egress 감사 패턴(메타데이터만) 재사용. 사이드카(8001) egress 감사와 별개 트랙. |

### B.3 UX / 확인 카드
- AI가 대화 중 write 의도를 감지하면 **답변 + 미리보기 카드**를 함께 낸다.
- 카드 = 정규화 필드(제목/상태/카테고리/대상 시트 라벨) + [실행]/[취소].
- **확인 전까지 8000 변경 0** — 자동 검증 가능한 안전선.
- 실행 성공 → 시스템 버블 "✓ 생성됨" + 결과 딥링크(이슈뷰/작업뷰 열기). 실패(403/400) → 카드에 에러, 그 외 영향 0.
- 부족 필드(예: 담당자 누락)는 카드 대신 AI가 **되물음**.

### B.4 안전 (P2)
- **격리 불변식:** 8001→8000 POST/PATCH **0건**(AST/정적 검사로 강제). 사이드카 write 권한 없음. `src/ai`는 기존 `src/api` write 함수만 호출(신규 write 경로 없음).
- **RBAC:** 실행이 기존 8000 라우트 → `require_role(편집자)` 그대로. 뷰어 → 403 + [실행] 버튼 UI disabled(canEdit 스레딩 재사용).
- **감사:** 확인된 액션은 origin=ai_chat로 감사로그에 기록.
- **휴먼인더루프:** AI 단독 write 경로 없음(제안만). 카드 확인이 유일한 실행 트리거.

### B.5 검증 / 테스트
- **사이드카 pytest:** 액션 툴이 올바른 제안 스펙 생성 · **8000 write 호출 0**(격리 테스트: AST/respx로 POST/PATCH 미발생 확인).
- **프론트 vitest:** ActionCard 렌더 · [실행]이 기존 API 호출 · [취소] 폐기 · 뷰어 disabled.
- **백엔드 pytest:** 액션 감사 기록 · origin 필드.
- **디바이스 e2e:** 실 gpt-5.5가 이슈/상태변경/작업 제안 → 확인 실행 → Issues/Tasks 뷰 반영 · 감사 기록 · 뷰어 차단 · 확인 전 서버 diff 0.

### B.6 Done-When (검증 기준)
1. 이슈 생성·상태변경·작업 생성을 AI가 제안하고 **카드 [실행] 시에만** 반영 (확인 전 서버 diff 0).
2. 사이드카 write 0 (격리 불변식 유지).
3. 뷰어 실행 차단 (403 + UI disabled).
4. 확인된 액션 감사로그 기록(actor·action·target·origin·시각).
5. 실행 결과 딥링크로 열림.
6. 회귀 0 (vitest·backend pytest·사이드카 pytest·build GREEN).

### B.7 범위 밖 (의도적, Out of Scope)
- 파일 업로드(챗 첨부 UX) · 이메일 발송 · MCP 노출 · 멀티유저 실인증 → P3~P5.
- 액션 되돌리기(undo)·배치 액션·마크업/측정 write → 후속.
- 이번 P1+P2는 **앱 내 단일사용자·핵심 3종 액션·휴먼인더루프**로 한정.
